2 워크플로우 실행 일정

⚡ 실시간 알림 (poll.yml)

*/30 * * * *

30분마다 실행. 새 글이 발견되면 즉시 Discord로 개별 알림 전송.

📊 주간 요약 (weekly.yml)

0 0 * * 1

매주 월요일 오전 9시 KST 실행. 지난 7일치 통계 + 키워드 분석을 Discord로 전송.

4 poll.yml — 실시간 알림 워크플로우

매 30분마다 아래 단계가 순서대로 실행됩니다.

  • 1
    트리거

    크론 */30 * * * * — 매 시간 0분, 30분에 자동 실행. workflow_dispatch로 수동 실행도 가능.

  • 2
    코드 체크아웃

    actions/checkout@v4 — main 브랜치 전체(seen.json 포함)를 Ubuntu 러너에 복사.

  • 3
    Python 3.11 설치 + pip 캐시 복원

    actions/cache@v4로 requirements.txt 해시를 키 삼아 캐시 복원. 의존성이 같으면 설치 생략.

  • 4
    poll.py 실행

    시크릿 DISCORD_WEBHOOK_URL, GITHUB_TOKEN_CUSTOM 주입 후 실행.

  • 5
    변경 파일 커밋 & 푸시

    seen.json(항상), HISTORY.md(새 항목 발견 시)를 스테이징. 변경분이 있으면 커밋 후 push. push 실패 시 pull —rebase 후 재시도.

# 커밋/푸시 핵심 로직
git add seen.json
[ -f HISTORY.md ] && git add HISTORY.md || true
git diff --staged --quiet || git commit -m "chore: update seen.json and HISTORY.md"
git push origin main || (git pull --rebase origin main && git push origin main)

5 weekly.yml — 주간 요약 워크플로우

매주 월요일 오전 9시 KST(UTC 00:00)에 실행됩니다.

  • 1
    트리거

    크론 0 0 * * 1 — 매주 월요일 00:00 UTC = KST 오전 9시. workflow_dispatch로 수동 실행 가능.

  • 2
    코드 체크아웃 + Python + pip 캐시

    poll.yml과 동일한 환경 설정.

  • 3
    weekly.py 실행

    지난 7일치 데이터를 수집·분석하여 Discord에 주간 요약 전송. seen.json을 수정하지 않으므로 커밋 단계 없음.
