3 GitHub Actions 실행 순서 (poll.yml)

매 30분마다 아래 단계가 순서대로 실행됩니다.

  • 1
    트리거: 스케줄 또는 수동

    크론 표현식 */30 * * * * — 매 시간 0분, 30분에 자동 실행됩니다. workflow_dispatch로 GitHub UI에서 수동 실행도 가능합니다.

  • 2
    코드 체크아웃

    actions/checkout@v4 — 레포의 현재 main 브랜치를 Ubuntu 러너에 복사합니다. seen.json도 이 시점에 함께 내려옵니다.

  • 3
    Python 3.11 설치 + pip 캐시 복원

    actions/cache@v4로 requirements.txt의 해시를 키 삼아 pip 캐시를 복원합니다. 의존성이 변경되지 않았으면 설치를 건너뜁니다.

  • 4
    poll.py 실행

    핵심 단계. 환경변수로 DISCORD_WEBHOOK_URL, GITHUB_TOKEN_CUSTOM 두 시크릿을 주입한 뒤 실행합니다.

  • 5
    seen.json 커밋 & 푸시

    새 항목이 있었을 경우 seen.json이 변경되어 있습니다. git pull --rebase 후 변경분을 커밋하고 main에 push합니다. 변경이 없으면 커밋하지 않습니다.

# poll.yml 핵심 부분
on:
  schedule:
    - cron: '*/30 * * * *'   # 30분마다
  workflow_dispatch:           # 수동 실행도 허용

steps:
  - run: python poll.py
    env:
      DISCORD_WEBHOOK_URL: ${{ secrets.DISCORD_WEBHOOK_URL }}
      GITHUB_TOKEN_CUSTOM: ${{ secrets.GITHUB_TOKEN_CUSTOM }}

  - run: |
      git pull --rebase origin main   # 충돌 방지
      git add seen.json
      git diff --staged --quiet || git commit -m "chore: update seen.json"
      git push

for repo in REPOS:                          # 레포 3개를 순회

    for issue in resp.json():
        if "pull_request" in issue:

        title = issue["title"].lower()
        body  = issue["body"].lower()
        if keyword in title or keyword in body: # "batch" 포함 여부 확인

인증 처리: GITHUB_TOKEN_CUSTOM 시크릿이 설정되어 있으면 인증 헤더를 추가합니다. 미설정 시 시간당 60회, 설정 시 시간당 5,000회 API 요청이 허용됩니다. 현재 30분마다 3개 레포 = 시간당 6회 요청이므로 미설정 상태에서도 여유 있습니다.

for feed_name, rss_url in RSS_FEEDS:
    feed = feedparser.parse(rss_url)         # RSS XML 파싱
    if feed.bozo and not feed.entries:
        continue                             # 파싱 실패 감지 후 건너뜀

feed.bozo란? feedparser가 RSS 파싱 중 오류를 감지하면 bozo=True로 표시합니다. entries가 비어 있을 때만 오류로 처리하여, 부분 파싱된 항목은 계속 활용합니다.

8 환경변수 & GitHub Secrets

변수명설정 위치필수용도
DISCORD_WEBHOOK_URL GitHub Secrets 필수 Discord 채널 웹훅 URL. 없으면 알림이 전송되지 않습니다.
GITHUB_TOKEN_CUSTOM GitHub Secrets 선택 GitHub API 인증 토큰. 없으면 시간당 60회 제한, 있으면 5,000회.
주의: 기본 GITHUB_TOKEN은 Actions에서 자동 제공되지만, 이 프로젝트는 별도의 GITHUB_TOKEN_CUSTOM을 사용합니다. Settings → Secrets and variables → Actions에서 등록해야 합니다.
